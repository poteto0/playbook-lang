# 技術選定: Rust

## 選定理由

### 1. WebAssembly (Wasm) との親和性
Rust は `wasm-bindgen` をはじめとする Wasm エコシステムが非常に成熟しています。これにより、以下のメリットが得られます。
- **クライアントサイド実行**: `playbook-lang` の解析・描画ロジックをブラウザ（Frontend）上で直接実行できます。これにより、サーバーへのリクエストなしで、ユーザー入力に対する瞬時のリアルタイムプレビューが可能になります。
- **ポータビリティ**: 作成したロジックは Frontend だけでなく、Cloudflare Workers などの Edge 環境や、一般的なサーバーサイドでも同様に動作します。

### 2. 言語処理（Parser）への適性
- **型安全性と堅牢性**: 独自言語のコンパイラやパーサーを作成する際、Rust の強力な型システムとメモリ管理はバグを未然に防ぐのに役立ちます。
- **パフォーマンス**: テキスト解析と SVG 生成は CPU 負荷がかかる処理ですが、Rust の高いパフォーマンスにより、低遅延での処理が可能です。
- **ライブラリ**: `nom`, `pest`, `chumsky` など、高品質なパーサーコンビネータライブラリが利用可能です。

### 3. プロジェクト構成への適合
- Frontend が `tanstackStart` + `Cloudflare Worker` という構成であるため、Rust で生成した Wasm モジュールを npm パッケージとして取り込む構成は非常にスムーズに統合できます。

## 構成案

- **言語**: Rust
- **ビルドターゲット**: WebAssembly (`wasm32-unknown-unknown`)
- **ツール**: `wasm-pack` (npm パッケージ生成用)
- **主要クレート候補**:
    - Parser: `nom` または `pest`
    - SVG生成: `svg` または 文字列操作による生成
